### C. 訪談大綱（Interview Guide）

#### 角色 1

| 欄位           | 內容                                                                   |
| -------------- | ---------------------------------------------------------------------- |
| 受訪角色       | 社團活動負責人                                                         |
| 角色與系統關係 | 活動申請方、系統主要操作者                                             |
| 訪談目的       | 挖掘紙本流程中的行政痛點，並確認「場地申請」的必要欄位與操作體驗需求。 |

| 題號 | 訪談問題                                                             | 想確認的需求或風險                     |
| ---- | -------------------------------------------------------------------- | -------------------------------------- |
| 1    | 目前申請一個活動，從準備到獲得核准，平均要耗時多久？                 | 量化行政延遲的痛點。                   |
| 2    | 在紙本申請中最容易出錯或遺漏的資訊是什麼？                           | 確認系統表單需強制填寫的欄位（需求）。 |
| 3    | 若改為線上申請，你最擔心無法解決的問題是什麼？                       | 預先發現潛在的抗拒因素與技術風險。     |
| 4    | 對於活動報名，你希望系統直接提供哪些數據統計（如：系級、人數）？     | 確認後台報表的需求範圍。               |
| 5    | 你希望在申請過程中，系統能主動提醒什麼資訊（如：場地衝突、截止日）？ | 定義自動化通知與警示的邏輯             |
| 6    | 有沒有哪一項活動相關的行政作業，是你覺得「絕對不能數位化」的？       | 確認開發範圍與隱藏的法律/規章邊界。    |

#### 角色 2

| 欄位           | 內容                                                     |
| -------------- | -------------------------------------------------------- |
| 受訪角色       | 課外活動組老師                                           |
| 角色與系統關係 | 系統審核者、行政決策者                                   |
| 訪談目的       | 確認審核邏輯、場地規範與系統權限設定，確保系統符合校規。 |

| 題號 | 訪談問題                                                                   | 想確認的需求或風險                          |
| ---- | -------------------------------------------------------------------------- | ------------------------------------------- |
| 1    | 在核准活動時，你最優先考量的條件有哪些？（如：場地容量、社團歷史紀錄）     | 建立審核流程的邏輯判斷標準。                |
| 2    | 目前審核過程中，最常發生「場地衝突」的時段或原因為何？                     | 確認系統防衝突演算法的關鍵參數。            |
| 3    | 對於社團申請，你是否有什麼「非硬性規定」的偏好或建議？                     | 了解系統彈性設定的需求。                    |
| 4    | 你是否需要針對特定社團或活動，進行「特殊權限」的加註或審核？               | 確認權限系統是否需要支援特殊狀況處理。      |
| 5    | 系統顯示的審核結果通知，你認為哪種狀態（如：核准、駁回、待補件）最為關鍵？ | 定義系統狀態機 (State Machine) 的轉換需求。 |
| 6    | 對於資料安全性，您最擔心什麼類型的資訊在平台上洩露？                       | 確認資安與隱私保護的優先級（風險管控）。    |
